---
title: Dataset Endpoints
description: Manage dataset configurations and lifecycle
---

Dataset endpoints allow you to register, retrieve, and manage dataset configurations in Amp. Datasets define how blockchain data is extracted and transformed.

## List All Datasets

<api method="GET" endpoint="/datasets" />

Returns all registered datasets across all namespaces with their version information.

### Response

<ResponseField name="datasets" type="array" required>
  List of all datasets across all namespaces
  
  <Expandable title="Dataset Summary">
    <ResponseField name="namespace" type="string" required>
      Dataset namespace
    </ResponseField>
    <ResponseField name="name" type="string" required>
      Dataset name
    </ResponseField>
    <ResponseField name="latest_version" type="string">
      Latest semantic version (if any)
    </ResponseField>
    <ResponseField name="versions" type="array" required>
      All semantic versions (sorted descending)
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

```bash
curl http://localhost:1610/datasets
```

### Example Response

```json
{
  "datasets": [
    {
      "namespace": "eth",
      "name": "blocks",
      "latest_version": "1.2.0",
      "versions": ["1.2.0", "1.1.0", "1.0.0"]
    }
  ]
}
```

## Register Dataset

<api method="POST" endpoint="/datasets" />

Registers a new dataset configuration in the local registry.

<Note>
  This endpoint only registers datasets and does NOT schedule data extraction. To extract data after registration, call the deploy endpoint.
</Note>

### Request Body

<ParamField body="namespace" type="string" required>
  Namespace for the dataset (validated identifier format)
</ParamField>

<ParamField body="name" type="string" required>
  Name of the dataset to be registered (validated identifier format)
</ParamField>

<ParamField body="manifest" type="object | string" required>
  Either a manifest hash (64-char hex string) or full manifest JSON content
</ParamField>

<ParamField body="version" type="string">
  Optional semantic version (e.g., "1.0.0"). If omitted, only the "dev" tag is updated.
</ParamField>

### Example Request

```bash
curl -X POST http://localhost:1610/datasets \
  -H "Content-Type: application/json" \
  -d '{
    "namespace": "eth",
    "name": "blocks",
    "version": "1.0.0",
    "manifest": {
      "kind": "evm-rpc",
      "network": "mainnet",
      "provider": "alchemy-mainnet",
      "start_block": 0
    }
  }'
```

### Response Codes

- **201 Created** - Dataset successfully registered
- **400 Bad Request** - Invalid dataset name, version, or manifest
- **500 Internal Server Error** - Database or object store error

## Get Dataset by Revision

<api method="GET" endpoint="/datasets/{namespace}/{name}/versions/{revision}" />

Returns detailed dataset information for the specified revision.

### Path Parameters

<ParamField path="namespace" type="string" required>
  Dataset namespace
</ParamField>

<ParamField path="name" type="string" required>
  Dataset name
</ParamField>

<ParamField path="revision" type="string" required>
  Revision - can be a version (e.g., "1.2.3"), hash, "latest", or "dev"
</ParamField>

### Response

<ResponseField name="namespace" type="string" required>
  Dataset namespace
</ResponseField>

<ResponseField name="name" type="string" required>
  Dataset name
</ResponseField>

<ResponseField name="revision" type="string" required>
  Revision requested
</ResponseField>

<ResponseField name="manifest_hash" type="string" required>
  SHA256 manifest hash
</ResponseField>

<ResponseField name="kind" type="string" required>
  Dataset kind (e.g., "evm-rpc", "firehose", "manifest")
</ResponseField>

<ResponseField name="start_block" type="integer" required>
  Starting block number
</ResponseField>

<ResponseField name="finalized_blocks_only" type="boolean" required>
  Whether to extract only finalized blocks
</ResponseField>

<ResponseField name="tables" type="array" required>
  List of table names in the dataset
</ResponseField>

### Example Request

```bash
curl http://localhost:1610/datasets/eth/blocks/versions/latest
```

### Example Response

```json
{
  "namespace": "eth",
  "name": "blocks",
  "revision": "latest",
  "manifest_hash": "8b065bde9c1a2f3e4d5c6b7a8e9f0a1b2c3d4e5f6a7b8c9d0e1f2a3b4c5d6e7f8",
  "kind": "evm-rpc",
  "start_block": 0,
  "finalized_blocks_only": false,
  "tables": ["blocks", "transactions", "logs"]
}
```

## Deploy Dataset

<api method="POST" endpoint="/datasets/{namespace}/{name}/versions/{revision}/deploy" />

Schedules a data extraction job for the specified dataset revision.

### Path Parameters

<ParamField path="namespace" type="string" required>
  Dataset namespace
</ParamField>

<ParamField path="name" type="string" required>
  Dataset name
</ParamField>

<ParamField path="revision" type="string" required>
  Revision (version, hash, "latest", or "dev")
</ParamField>

### Request Body

<ParamField body="end_block" type="string | number | null">
  End block configuration:
  - `null` (default): Continuous dumping
  - `"latest"`: Stop at latest block
  - `<number>`: Stop at specific block
  - `<negative>`: Stop N blocks before latest
</ParamField>

<ParamField body="parallelism" type="integer">
  Number of parallel workers (default: 1, only for raw datasets)
</ParamField>

<ParamField body="worker_id" type="string">
  Optional worker selector - exact ID or glob pattern
</ParamField>

### Response

<ResponseField name="job_id" type="integer" required>
  The ID of the scheduled extraction job
</ResponseField>

### Example Request

```bash
curl -X POST http://localhost:1610/datasets/eth/blocks/versions/latest/deploy \
  -H "Content-Type: application/json" \
  -d '{
    "end_block": "latest",
    "parallelism": 4
  }'
```

### Example Response

```json
{
  "job_id": 12345
}
```

## List Dataset Versions

<api method="GET" endpoint="/datasets/{namespace}/{name}/versions" />

Returns all versions for a dataset with their metadata.

### Path Parameters

<ParamField path="namespace" type="string" required>
  Dataset namespace
</ParamField>

<ParamField path="name" type="string" required>
  Dataset name
</ParamField>

### Response

<ResponseField name="namespace" type="string" required>
  Dataset namespace
</ResponseField>

<ResponseField name="name" type="string" required>
  Dataset name
</ResponseField>

<ResponseField name="versions" type="array" required>
  List of semantic versions (sorted descending)
  
  <Expandable title="Version Info">
    <ResponseField name="version" type="string" required>
      Semantic version
    </ResponseField>
    <ResponseField name="manifest_hash" type="string" required>
      Manifest hash for this version
    </ResponseField>
    <ResponseField name="created_at" type="string" required>
      When this version was created
    </ResponseField>
    <ResponseField name="updated_at" type="string" required>
      When this version was last updated
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="special_tags" type="object" required>
  Special tags (latest and dev)
  
  <Expandable title="Special Tags">
    <ResponseField name="latest" type="string">
      Latest semantic version (if any)
    </ResponseField>
    <ResponseField name="dev" type="string">
      Dev tag pointing to manifest hash (if any)
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

```bash
curl http://localhost:1610/datasets/eth/blocks/versions
```

## Delete Dataset

<api method="DELETE" endpoint="/datasets/{namespace}/{name}" />

Removes all manifest links and version tags for a dataset. This operation is fully idempotent.

### Path Parameters

<ParamField path="namespace" type="string" required>
  Dataset namespace
</ParamField>

<ParamField path="name" type="string" required>
  Dataset name
</ParamField>

### Response Codes

- **204 No Content** - Dataset successfully deleted (or didn't exist)
- **400 Bad Request** - Invalid path parameters
- **500 Internal Server Error** - Database operation error

### Example Request

```bash
curl -X DELETE http://localhost:1610/datasets/eth/blocks
```

## Delete Dataset Version

<api method="DELETE" endpoint="/datasets/{namespace}/{name}/versions/{version}" />

Removes a semantic version tag from a dataset. Cannot delete the "latest" version.

### Path Parameters

<ParamField path="namespace" type="string" required>
  Dataset namespace
</ParamField>

<ParamField path="name" type="string" required>
  Dataset name
</ParamField>

<ParamField path="version" type="string" required>
  Semantic version to delete (e.g., "1.2.3")
</ParamField>

### Response Codes

- **204 No Content** - Version successfully deleted
- **400 Bad Request** - Invalid parameters or attempting to delete "latest"
- **500 Internal Server Error** - Database operation error

### Example Request

```bash
curl -X DELETE http://localhost:1610/datasets/eth/blocks/versions/1.0.0
```
